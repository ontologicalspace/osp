## What this artifact is

This artifact supports the verification claims of OSP Paper 3. It contains **evidence outputs** — deterministic snapshots generated by the named tests in the OSP `osp-core` crate — **not the private repository source code**. The repository was private at submission time; this pack preserves the verification evidence surface so that a reader can independently inspect the frozen artifacts the paper cites.

## What this artifact is not

- It is **not** the OSP source code. The Rust workspace (types, invariants, gates, tests) remains private at this stage of the project.
- It is **not** a re-runnable test suite. The files are *frozen snapshots*; regenerating them requires the private source.
- It is **not** a benchmark. Paper 3 is a verification paper; these artifacts are verification evidence, not performance measurements.

## Integrity

Each evidence file's sha256 is recorded in `MANIFEST.json` and in `run-metadata.json`. The sha256 values were computed over the repository's LF-normalized content (the canonical stored form). On a CRLF checkout (Windows), the working-tree sha256 will differ; the repository-form sha256 is authoritative.

## Reproduction (future)

When the OSP source becomes public, the frozen artifacts can be regenerated by:
```bash
git clone <osp-repo-at-frozen-commit>
cd osp
cargo test -p osp-core --test paper3_evidence -- --ignored --nocapture  # regenerate e2e + rejected
cargo test -p osp-core --test paper3_heldout -- --ignored --nocapture   # regenerate held-out
```
with `PAPER3_FREEZE=1` set. The regenerated files must match the sha256 values in `MANIFEST.json`.
